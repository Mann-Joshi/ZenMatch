## 2024-05-01 - [Mahjong Logic - Unified Collision Check]
**Learning:** In Mahjong Solitaire algorithms with overlapping fractional coordinates, checking for blockers via intermediate arrays (e.g., `getNearbyTiles`) causes massive GC pressure and redundant nested iteration. It is also risky to try to inline them blindly as `candidate.x === tile.x` without properly considering fractional scale radius and depth.
**Action:** Always favor a single unified grid traversal (`checkBlockers`) that performs early exits (`if (blockedAbove && blockedLeft && blockedRight)`) and accesses spatial bucket-maps directly without allocating intermediary arrays. When optimizing, encapsulate the unified traversal in a clean helper rather than leaving giant nested loops in the core loop to maintain readability.
