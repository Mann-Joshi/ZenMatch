## 2024-05-23 - Single-Pass Grid Traversal Over Buckets
**Learning:** For Mahjong tile collision and blocker checks (like `computeFreeTiles`), performing separate function calls that each iterate over the grid and allocate candidate arrays via spreading (e.g. `candidates.push(...bucket)`) creates significant overhead. Consolidating top, left, and right blocker checks into a single unified grid traversal that accesses tile buckets directly avoids multiple iterations and intermediate array allocations, resulting in substantial execution time improvements in hot paths without sacrificing correctness.
**Action:** Always prefer a unified single-pass nested loop traversal over grid buckets for multiple spatial checks instead of multiple single-responsibility but redundant loops, especially when the check is called frequently like in hint or pair generation.
