## 2024-05-18 - [Unified Mahjong Grid Traversal Optimization]
**Learning:** In heavily accessed spatial computations like checking Mahjong tile freeness (`computeFreeTiles`), avoiding intermediate array allocations (e.g., using `push(...bucket)` into a temporary array and then iterating over it) significantly reduces garbage collection overhead and execution time. Furthermore, combining independent multi-directional spatial checks (`hasTopBlocker` and `hasSideBlocker`) into a single unified spatial iteration prevents redundant loop execution.
**Action:** When working on coordinate or grid-based collision detection that is called frequently (especially in a loop or recursively like in `getHintPair`), prioritize direct, unified single-pass iterations over the data structure over breaking the logic into multiple clean but overlapping mapping passes.
